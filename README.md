<h2>Part 5. Использование команды sudo</h2>

![image](https://github.com/TiroBite/tirobite/assets/102403027/515fb2a4-b212-4f26-9581-0890a5bd825b)


Имя "sudo" расшифровывается как "Super User Do" или "Substitute User Do". Основная цель sudo состоит в предоставлении временного доступа к привилегиям суперпользователя (root) конкретному пользователю, не предоставляя ему полные привилегии root на постоянной основе. Это помогает уменьшить риск ошибок и повышает безопасность системы, поскольку пользователь должен явно запрашивать привилегии root для выполнения определенной команды.
Все команды, выполненные с его помощью, обычно записываются в журнал системы. Это позволяет администраторам системы отслеживать, кто и когда выполнял привилегированные операции.

<h2>Part 8. Установка и базовая настройка сервиса SSHD</h2>

Требуемые шаги:
1. Установка SSH: sudo apt install openssh-server -y
2. После установки службы SSH будет автоматически запущена. Вы можете проверить статус службы с помощью: sudo service sshd status
3. Чтобы настроить автозапуск, выполните: sudo systemctl enable ssh
4. Используя команду ps, чтобы показать наличие процесса sshd. Для этого к команде нужно подобрать ключ: ps -e | grep sshd

* Флаг -е позволяет увидеть все процессы запущенные в системы

![image](https://github.com/TiroBite/tirobite/assets/102403027/0a2a3a0d-e6ac-4ff0-90bd-78a84c8724e1)

Для netstat -tan каждый флаг означает:
<li>-t: отображает только TCP-порты.</li>
<li>-a: отображает все активные соединения, включая слушающие порты и подключения.</li>
<li>-n: отображает порты и адреса в числовом формате</li>

Значение столбцов:
<li>Proto (Protocol): Этот столбец показывает используемый протокол, такой как TCP.</li>
<li>Recv-Q и Send-Q (Receive Queue и Send Queue): Эти столбцы отображают длину очереди ожидания для приема и отправки данных соответственно.</li>
<li>Local Address (Локальный адрес): Этот столбец указывает на локальный IP-адрес и порт. Например, "0.0.0.0:2022" означает, что сервер слушает на всех интерфейсах (0.0.0.0) на порту 2022.</li>
<li>Foreign Address (Внешний адрес): Этот столбец указывает на удаленный IP-адрес и порт, к которому установлено соединение.</li>
<li>State (Состояние): Этот столбец указывает на текущее состояние соединения. Например, "ESTABLISHED" означает, что соединение установлено.</li>
